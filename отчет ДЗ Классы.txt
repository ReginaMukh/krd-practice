### Задание№2 ДЗ по лекции "Классы и модули"
###В отчете расписать, понимание работы класса Router, причины, по которым используется extend для модуля Resource

	Класс Router - для указания, с каким путем надо отправить запрос, чтобы действовал контроллер.
	Объектом класса Router  является некий хэш @routes, который будет хранить  ссылки на методы контроллера для последующего их вызова.
	В методе resources описаны стандартные пути управления определенной моделью (postом/commentом)
	В данном классе создается путь (routes) для поста, и соответсвующий котроллер для  него. После того как путь (routes) принимает запрос - ключ-значение 'posts', он отправляет его на дальнейшую обработку методом PostsController.connection

	PostsController миксуется с модулью Resource методом extend, по причине того, что обратиться к методу данного модуля  необходимо классу , чтобы он был доступен каждому объекту данного класса, а не только определенному экземпляру. Чтобы не расписывать метод connection для каждого класса-контроллера, его вынесли в отдельный модуль и можно  миксовать при необходимости .
